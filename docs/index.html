Introduction

What is Angular?
Angular is a platform that facilitates building applications with the web.
It is a framework for building client applications in HTML and either JavaScript or a language like TypeScript that compiles to JavaScript.
Angular is written in TypeScript. It implements core and optional functionality as a set of TypeScript libraries that you import into your apps.
It is modular and uses dependency injection to make your code modular, reusable, and efficient.

In this project I will be trying to build angular from scratch in order to better understand how it works and how it is built.
I tried to use the same file structure and naming conventions as much as possible.
I also tried to develop the core functionality of angular using the same design patterns as angular.

This is a learning project and not meant to be used in production.

Services

Services are used to manage the state of the application, they handle the data and the logic of the application.
Dependency Injection is used to inject services into components.
A services container is used to manage the services and the dependency injection, by keeping a unique instance of each service.

Directives

Directives are used to manipulate the DOM, in a way similar to angular directives.
to create a new directive you need to create a class that extends that is decorated with the @Directive decorator.

the @Directive decorator takes an object with the selector property and providers property.

  • selector: string - the selector is the name of the directive, it can be used as an attribute or as an element.
  • providers: Array - the providers property is an array of services that will be injected into the directive.
When the directive is created, the constructor is called and the services are injected into the directive. Then it is associated with the DOM element that it is attached to using the selector.

Components

Components are used to create the view of the application, they are a combination of a template and a controller.
To create a new component you need to create a class that is decorated with the @Component decorator.
The @Component decorator takes an object with the selector property, template property, and the providers property

  • selector: string - the selector is the name of the component, it can be used as an attribute or as an element.
  • template: string - the template property is the template of the component, it is a string containing the HTML structure. Both interpolation and events can be used in the HTML template
  • providers: Array - the providers property is an array of services that will be injected into the component.
When the component is created, the constructor is called and the services are injected into the component. Then the template is rendered wherever the component's selector is used.

Change Detection

JS Proxy

Change detection is used to detect changes in the state of the application and update the view accordingly.
Change detection is done using a proxy object, whenever a property of the proxy object is accessed, the change detection algorithm is run.

Change Detector

To prevent excessive manipulation of the DOM and to improve performance, changes are registered in a queue and only the most recent change is kept.
Then the change detection algorithm is run when a digest function is called and update the view.

Zone.JS

Zone.JS is used to run the change detection algorithm when the state of the application changes.
it creates a zone where all functions calls are intercepted and can be modified.
This is where after every function call, the digest function of the change detector is called in order to update the view.
This allows to improve developer experience by not having to manually call the digest function.
